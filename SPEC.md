# VE機能系統図ツール — 仕様書 (SPEC.md)

## 1. 概要

### 目的
バリューエンジニアリング（VE）の手法である「機能系統図」をブラウザ上で作成・編集・保存できるWebアプリ。機能を目的と手段の関係で整理し、視覚的に図として表現する。

### 対象ユーザー
- VEコンサルタント・担当者
- 製品開発・原価管理部門
- VEを学習中の個人

## 3. サイドバー仕様

### 3-1. 折りたたみ動作

| 状態 | 幅 | 表示内容 |
|------|----|---------|
| 展開 | 320px | テーブル + Excel アップローダー + アフィリエイトパネル |
| 折りたたみ | 48px | トグルアイコンのみ |

- `≡` ボタンクリックで状態トグル
- 状態はローカルstateで管理（localStorageに保存しない）

### 3-2. 構成要素×機能テーブル

#### テーブル構造

```
┌──────────────┬──────────────────────────┬────┐
│ 構成要素      │ 機能                      │    │ ← ヘッダー行
├──┬───────────┼──────────────────────────┼────┤
│🎨│ [テキスト] │ [テキスト]               │ ✕ │ ← データ行
├──┴───────────┼──────────────────────────┼────┤
│  + 行を追加   │                          │    │
└──────────────┴──────────────────────────┴────┘
```

#### ComponentRow 型
```typescript
interface ComponentRow {
  id: string;           // nanoid()で生成
  component: string;    // 構成要素テキスト
  functionText: string; // 機能テキスト
  color: string;        // hex色コード (e.g. "#3B82F6")
}
```

#### 操作
- **行追加**: `+ 行を追加` ボタン → 空行を末尾に追加
- **テキスト編集**: 各セルは `<input type="text">` で直接編集
- **色変更**: 左端のカラーサークル（🎨）クリックで `<input type="color">` を開く
- **行削除**: `✕` ボタンで行削除（確認なし）。関連ノードの色はデフォルト色に戻す

#### 色とノードの連動
- 同じ `component` 文字列を持つノードは同じ色を持つ
- 色変更時は対応するノードすべてに即座に反映

### 3-3. Excel アップローダー

テーブル上部（または下部）にドロップゾーンを配置:

┌──────────────────────────────────┐
│  📂 Excelファイルをドロップ       │

## 4. ダイアグラムキャンバス仕様

### 4-1. React Flow 設定

```typescript
<ReactFlow
  nodes={nodes}
  edges={edges}
  nodeTypes={nodeTypes}    // { functionNode: FunctionNode }
  edgeTypes={edgeTypes}    // { ortho: OrthoEdge }
  defaultEdgeOptions={{ type: 'ortho' }}
  snapToGrid={true}
  snapGrid={[20, 20]}
  fitView
/>
```

- **背景**: `<Background variant="dots" gap={20} />`
- **コントロール**: `<Controls />` (左下)
- **ミニマップ**: `<MiniMap />` (右下、オプション)
- **スナップグリッド**: 20px単位

### 4-2. カスタムノード: FunctionNode

#### 見た目
```
 F1-1                   ← 機能番号 (小さいフォント, ノード上部)
┌──────────────────────┐
│                      │  ← 背景色はcomponentに対応
│   商品を導入させる     │
│                      │
└──────────────────────┘
```

#### FunctionNodeData 型
```typescript
interface FunctionNodeData {
  label: string;         // 機能テキスト (例: "商品を導入させる")
  number: string;        // 機能番号 (例: "F1-1") ← 自動計算
  color: string;         // 背景色 hex
  componentId: string;   // 紐付く ComponentRow の id
}
```

#### ハンドル
- **左ハンドル** (`Position.Left`): 接続の入力側（Target）
- **右ハンドル** (`Position.Right`): 接続の出力側（Source）
- ハンドルは視覚的に小さな円（ホバー時に表示）

#### スタイル
- 幅: 160px 固定 / 高さ: auto (最小48px)
- 角丸: `border-radius: 6px`
- テキスト色: 背景色のコントラストに応じて自動切替（白/黒）
- 番号: `font-size: 10px`, ノード上端の外側に表示（`position: absolute; top: -16px`）
- ノードを重ねる場合: `z-index` をドラッグ時に最前面に（React Flow デフォルト動作）

### 4-3. カスタムエッジ: OrthoEdge

- 基底: React Flow `SmoothStepEdge`（`borderRadius: 4`）
- 接続方向: 必ず右→左（source右ハンドル → target左ハンドル）
- 線スタイル: `stroke: #94a3b8`, `strokeWidth: 1.5`
- 先端マーカー: なし（ノードと線の接点のみ）
- 折れ点: ソースの右端から水平に出て、ターゲットの左端で折れる

### 4-4. 階層ゾーン

x座標でレベルを判定（ドラッグ後の位置を元に番号を再計算）:

| レベル | x 範囲 | 番号例 |
|--------|--------|-------|
| Level 1 | 0 〜 299 | F1, F2 |
| Level 2 | 300 〜 599 | F1-1, F1-2 |
| Level 3 | 600 〜 899 | F1-1-1, F1-1-2 |
| Level 4 | 900 〜 1199 | F1-1-1-1 |

- ゾーン幅は定数 `ZONE_WIDTH = 300` で管理
- 最大階層数: 4（拡張可能な設計にすること）

### 4-5. 番号自動付与ロジック

```
1. ルートノード判定: 入力エッジ(target)がないノード = Level 1
2. Level 1 ノードをy座標昇順でソート → F1, F2, F3...
3. 各ノードの子（出力エッジのtarget）をy座標昇順でソート → F1-1, F1-2...
4. 深さ優先再帰で番号を付与
5. エッジ追加/削除/ノード移動のたびにすべて再計算
```

再計算タイミング: `onNodesChange`, `onEdgesChange`, `onConnect` のイベントハンドラ内

### 4-6. ノード操作

| 操作 | 方法 |
|------|------|
| ノード追加 | ツールバー「＋ノード追加」ボタン → キャンバス中央に新規追加 |
| ノード移動 | ドラッグ（スナップグリッド: 20px） |
| ノード選択 | クリックで選択、Shift+クリックで複数選択 |
| ノード削除 | 選択後 Delete/Backspace キー、またはツールバー「削除」 |
| ノード編集 | ダブルクリックでラベルをインライン編集 |
| 接続 | ツールバー「接続モード」で右ハンドルからドラッグ |
| 物理スタック | 任意の位置に自由配置可（同位置の重複を許可） |

## 6. 状態管理 (Zustand Store)

### diagramStore.ts

```typescript
interface DiagramState {
  // テーブルデータ
  tableRows: ComponentRow[];

  // React Flow データ
  nodes: Node<FunctionNodeData>[];
  edges: Edge[];

  // UI
  selectedNodeIds: string[];

  // アクション: テーブル
  addTableRow: () => void;
  updateTableRow: (id: string, patch: Partial<ComponentRow>) => void;
  deleteTableRow: (id: string) => void;

  // アクション: ノード/エッジ
  onNodesChange: OnNodesChange;
  onEdgesChange: OnEdgesChange;
  onConnect: OnConnect;
  addNode: (componentId: string) => void;
  deleteSelectedNodes: () => void;

  // アクション: インポート
  importFromExcel: (rows: ComponentRow[]) => void;

  // アクション: エクスポート
  exportToPng: (flowRef: RefObject<HTMLDivElement>) => Promise<void>;

  // 内部: 番号再計算
  recalculateNumbers: () => void;
}
```

### localStorage 永続化

`zustand/middleware` の `persist` を使用:
```typescript
persist(store, {
  name: 'vft-diagram-v1',
  // nodes と edges と tableRows のみ永続化
  partialize: (state) => ({
    tableRows: state.tableRows,
    nodes: state.nodes,
    edges: state.edges,
  }),
})
```

## 7. Excel インポート仕様

### 7-1. 対応ファイル
- `.xlsx`, `.xls`
- 最大ファイルサイズ: 5MB

### 7-2. 期待する列構成

| 列A | 列B |
|-----|-----|
| 構成要素 | 機能 |
| モーター | 電力を変換する |
| モーター | トルクを伝える |
| フレーム | 荷重を支える |

- 1行目はヘッダー行として読み飛ばす
- 列名は「構成要素」「機能」で検索（存在しない場合はA列・B列をフォールバック）

### 7-3. インポート処理フロー

```
1. SheetJS で .xlsx を読み込み、最初のシートをJSONに変換
2. 構成要素でグループ化
3. 各グループに順番に色を自動割り当て（パレット: 6色）
4. テーブル行を生成 → tableRows に追加（既存と重複する場合は確認ダイアログ）
5. ノードを自動配置:
   - 各行のノードを Level 2 の列（x=340）に配置
   - y 座標: 60px 間隔でソート
6. 番号再計算
```

### 7-4. 自動割り当てカラーパレット

```typescript
const DEFAULT_COLORS = [
  '#3B82F6', // blue
  '#06B6D4', // cyan
  '#10B981', // green

  '#8B5CF6', // violet
];
```

### 7-5. 列マッピングUI（オプション）

```
┌──────────────────────────────┐

## 8. PNG エクスポート仕様

### 処理フロー

```
1. html-to-image の toPng() を React Flow の DOMノードに対して実行
2. ダウンロード: a.href = dataUrl; a.download = `ve-diagram-${timestamp}.png`
3. オプション: 背景色を白に設定（デフォルトはダークグリッド背景）
```

### エクスポート設定（将来拡張用）
- 解像度: 2x（retina対応）
- 背景色: 白（`#FFFFFF`）
- パディング: 40px

## 9. アフィリエイトパネル仕様

サイドバー下部に折りたたみ可能なセクションとして配置:

```

─────────────────────

- 各カードにはアフィリエイトリンク（外部URL）を設定

- 初期は静的コンテンツ（ハードコード）

## 11. 非機能要件

- **対応ブラウザ**: Chrome/Edge 最新版（モバイル非対応）
- **パフォーマンス**: ノード100個までは快適に動作すること
- **アクセシビリティ**: キーボードナビゲーション（基本操作のみ）
- **セキュリティ**: 外部APIなし、XSS対策（Reactのデフォルト）
